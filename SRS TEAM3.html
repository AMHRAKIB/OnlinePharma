1. Introduction
1.1 Purpose
An online pharmacy e-commerce website. The purpose of this document is to provide overview of our e-commerce website. This website will be able to provide all sort of legal drugs through single get way using internet. Through this website dealers and customer will be able to buy medicines on emergency situation without visiting in person. Administration will enable system administrator to approve, maintain lists of drugs on pharmacy. The features of the application including the phone interface and web interface, the system server in use and the limitations of this application from the perspective of the users and the developers are also part of this document. This document is intended for both the user and the developer.

1.2 Document Conventions
The typographical conventions in this srs contains the standard referred to as IEEE Std 1003.1-2001. Each page contains a topic consisting of a sub topic written in bold convention subjecting to detailed requirement statement having its own priority. We use standard HTML for writing.

3. External Interface Requirements
3.1 User Interfaces:
 The design and layout of the website as well as the app need to be precise and naive to the user so that he/she can easily operate them. At the top right corner, there will be two options. These are:
1.Sign in : this will permit the user to enter username/email id and password to enter his/her account (already created before)
2.Create account : this option permits the new comer to fill up the necessary informations needed to create the account.
The account is created just to keep track of the activities of the user, i.e when and how many medicines the user ordered till now and the prescriptions which have been uploaded, etc. There will be a search bar at the top-mid portion of the page, which will be used to look for the medicines that the user wants to order. The user can type in the name of the medicines or can simply click on the mic icon beside it and pronounce the name and search. Due to this voice recognition system, not only people of rural areas but also people of urban areas will find it easy to use. And there will be also an option where the user can upload his/her prescriptions so the system can suggest medicines accordingly.

3.2 Hardware interfaces:
 Communication protocols are important as they cover authentication, error detection and correction. It is necessary for the software to have communication protocols, and since the software requires to be run on internet, Modem, WAN-LAN, etc. will be used.

5. Other Nonfunctional Requirements
5.1 Performance Requirements
 It should be able to give access to maximum user at the same time. And maximum customer, who will access should be able to have the same speed. And maximum user should get the same speed.

5.2 Safety Requirements
 The application should be work accordingly to the given prescription. As medicine will be shown as per written over the prescription. So, the application should have enough accuracy to scan the prescription. The application will not harm any stored data or outside data storage. And it will not also harm other web application.
The potential safety which will be considered mostly, it should be accessed by any smart handset. And categories of medicine should be showed related to that medicine which is required by user.

5.3 Security Requirements
 As it is a web based server system, most of the data are kept on server. So, there should be correctness of the claim of a user that owns a specific identity. Control of the user performance comes to first priority of any application. Controlling access when the users identity has been verified. There will right of the user and also limitation to access or use sensitive data.
Confidentiality is the most important safety things which are combined with identification of user, which will not be exchanged or enclosed to unauthorized parties.
Since all the data will be transferred on the web, system should also use an encryption and decryption mechanism only intended user can decode the data and work on the data.

5.4 Software Quality Attributes
 The design of the software is made easy accessible for both the user and the developer. It is well secured, easily usable and performed well. It has the opportunity to update it with data and have the option for maintaining avidity. It has scanning process, people can get there required medicine by scanning prescription.it will also help the people with voice search the easiest option. We tried our best to make it simple as possible. Because is medicine is our basic need. So people can access the application as easily as possible.

5.5 Business Rules
 Our software mainly built on keeping the concern of daily necessity of people. This software will be accessible by all users to make their life easier. We tried to build the software which has easy access with all the safety and security. Strict rules will be maintained for medicine supply so that no one get harm. The user will have easy ability to communicate. Scanning process will as accurate as possible. The voice search will be on Asian accent so normal people can access it.
