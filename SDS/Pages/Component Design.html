2.0. Component Design

Section 2.1. Description

Component design is the geometric layout of components. Geometry details have a major influence on the component reliability The purpose of component design is to define data structures, algorithms, interface characteristics, and communication mechanisms for each software component identified in the architectural design. Component design occurs after the data and architectural designs are entranched. The component design represents the software in a way that allows the designer to review it for faultlessness or preciseness and consistency, before it is built. The work product produced is a design forbeach software component, represented using graphical, tabular, or text-based notation.
